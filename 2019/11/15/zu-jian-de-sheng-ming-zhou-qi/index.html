组件的生命周期

组件的生命周期

created()

在data同层级下：

lifetimes:{
    created(){
      console.log("this.data.msg",this.data.msg);               //能获取到
      console.log("this.properties.item",this.properties.item); //获取不到

    }
  },

created()在组件实例刚刚被创建的时候执行，这里不能调用setData方法

attached()

在组件实例进入页面节点树的时候执行

可以获取this.properties.item

lifetimes:{
    attached(){
      console.log("this.data.msg",this.data.msg);               //能获取到
      console.log("this.properties.item",this.properties.items);//能获取到
    }
},

ready()

组件布局完成后执行

lifetimes:{
    ready(){
      console.log("this.data.msg",this.data.msg);               //能获取到
      console.log("this.properties.item",this.properties.items);//能获取到
    }
},

move()

组件被移动到节点树的另一个位置的时候执行

detached()

组件被从节点树上移除的时候触发

lifetimes:{
    detached(){
      console.log("detached");
    }
}

在父组件中接收自己定义的方法

在父组件的js中写这个删除事件

handleDelete(e){
    console.log(e.detail);
    let datas = this.data.arr;
    let result =  datas.filter(v=>{
      return v.id!=e.detail;
    })
    this.setData({
      arr:result
    })  
}

vant-ui的使用

在github下载vant-ui

放在自己的项目下解压

在项目下新建文件plugins,

再再目录下新建文件夹vant

将解压包中的dist文件夹下所有文件站粘贴到刚才新建的vant 文件夹下

需要用这个组件的json文件添加：

{
  "usingComponents": {
    "van-button":"/plugins/vant/button/index"
  }
}

wxml里可以使用了：

<van-button type="warning">警告按钮</van-button>
