组件化(重点)

Vue组件化思想

组件化

组件化提供了一种抽象，让我们可以开发出一个个独立可复用的小组件来构造我们的应用

任何的应用都会被抽象成一个组件树

组件化思想的应用

有了组件化的思想，我们在之后的开发中就要充分利用它

尽可能地将页面拆分成一个小的、可复用地组件

这样我们的代码更加方便组织和管理，扩展性也更强

注册组件的基本步骤

创建组件构造器

调用Vue.extend()方法创建组件构造器

传入template是自定义组件模板

    const cpnC = Vue.extend({
            template:`
            <div>
                <h2>我是标题</h2>
                <p>我是内容哈哈哈</p>
                <p>我是内容笑笑笑笑笑笑笑笑笑</p>
                <p>我是内容水水水水</p>
            </div>
            `
        })

注册组件

调用Vue.component()方法

调用Vue.component()方法是将刚才的租价能够早期注册为一个组件，并且给他起一个组件的标签名

所以需要传递两个参数：1、注册组件的标签名  2、组件构造器

//注册组件
Vue.component('my-cpn',cpnC)

使用组件

在Vue实例的作用范围内使用组件

比如下例的id="app"下才有效

<div id="app">
        <my-cpn></my-cpn>
        <my-cpn></my-cpn>
        <my-cpn></my-cpn>
</div>

全局组件和局部组件

全局组件

在Vue实例外注册的都是全局组件，只要在Vue的实例范围内使用都可以

调用Vue.component()方法

局部组件

在Vue实例内注册组件，这个组件只可以在自己的组件范围内使用

let app = new Vue({
            el: `#app`,
            data: {
                message: '你好啊'
            },
            components:{
                //cpn使用组件时的标签名
                cpn:cpnC
            }
        })
components:{
       //cpn使用组件时的标签名
       cpn:cpnC
}

上面的局部组件只可以在id="app"的实例范围内使用这个组件

父子组件

可以在组件内调用组件

例如我们创建了cpnC1 组件：

const cpnC1 = Vue.extend({
            template:`
            <div>
                <h2>我是标签1</h2>
                <p>我是内容，哈哈哈哈哈哈</p>
            </div>    
            `
        })

接着我们创建第二个组件cpnC2 ，在这个组件内我们想调用第一个组件,我们可以在第二个组件注册第一个组件：components : { cpn1 : cpnC1 }，这样就可以在第二个组件内使用组件一了

const cpnC2 = Vue.extend({
            template:`
            <div>
                <h2>我是标签2</h2>
                <p>我是内容，谢谢谢谢谢谢</p>
                <cpn1></cpn1>
            </div>    
            `,
            components:{
               cpn1:cpnC1 
            }
        })

同时我们使用组件二的时候也需要注册组件二才可以

注册组件语法糖

省去创建的过程，提供注册的语法糖

我们原来使用组件，首先通过const cpnC1 = Vue.extend({...}) 创建一个组件，然后注册这个组件Vue.component("cpn1", cpnC1)

我们可以通过语法糖省去第一步创建全局组件：

Vue.component("cpn1",{
            template:`
            <div>
                <h2>我是标签1</h2>
                <p>我是内容，哈哈哈哈哈哈</p>
            </div>    
            `
        })

创建局部组件如下：

let app = new Vue({
            el: `#app`,
            data: {
                message: '你好啊'
            },
            components: {
                'cpn1': {
                    template: `
                        <div>
                            <h2>我是标签1</h2>
                            <p>我是内容，哈哈哈哈哈哈</p>
                        </div>    
                        `
                }
            }
        })

组件模板的分离写法

script标签

类型必须是text/x-template

<script type="text/x-template" id="cpn">
        <div>
            <h2>我是标题</h2>
            <h2>我是内容哈哈哈哈</h2>
        </div>
</script>

然后注册组件：

//注册一个全局组件
Vue.component('cpn',{
    template:'#cpn'
})


let app = new Vue({
    el: `#app`,
    data: {
        message: '你好啊'
    }
})

然后在实例范围内调用组件就可以了：

<div id="app">
        <cpn></cpn>
</div>

template标签

<template id="cpn">
    <div>
        <h2>我是标题</h2>
        <h2>我是内容哈哈哈哈</h2>
    </div>
</template>

然后注册和调用同上

疑问？

1、组件内部可以访问Vue里的数据吗？

不能访问，组件是一个单独功能模块的封装，那么组件自己的数据放在哪呢？

组件对象里也有一个data属性(也可以有methods等属性)，只是这个data属性必须是一个函数，而且这个函数返回一个对象，对象内部保存数据

<template id="cpn">
    <div>
        <h2>组件内的数据：：{{title}}</h2>
    </div>
</template>

<script src="../js/vue.js"></script>
<script>
      Vue.component('cpn',{
          template:'#cpn',
          data(){
              return{
                  title:"我是组件内的数据"
              }
          }
      })
</script>

只有这样才能保存组件内的数据

2、组件内的data为什么必须是一个函数？

因为组件封装后就会是一个独立的单位，调用相同组件多次，内部数据由于是用函数保存的，用函数保存，每一次一个新的组件需要的数据都是一个新的地址保存各自的数据，所以不会污染。

data(){
    return{
        counter:0
    }
}

如果使用对象data保存数据，调用多次的时候，修改一个data，其他data数据也会被污染，因为调用对象的实质是调用了对象的地址，如果数据用对象保存，那么相同组件调用的都是同一个对象地址

data:{
    return{
        counter:0
    }
}

上面使用对象保存数据，就会报错

父子组件的通信

问题描述：

上一节中，我们知道子组件不能引用父组件或者Vue实例的数据的

但是我们在开发中确实需要一些数据展示在小组间中

这时候我们不会让小组件再发送一次网络请求，而是直接让父组件将数据传递给子组件

如何父子组件通信？

父传子：通过props向子组件传递数据

父组件的数据：

let app = new Vue({
    el: `#app`,
    data: {
        message: '你好啊',
        movies:['海王','海贼王','海尔兄弟','火影忍者']
    },
    components:{
        cpn
    }
})
props用数组接收数据

首先在props里定义接收数据的容器名字（注意这里不支持驼峰命名，因为v-bind不支持驼峰命名）

const cpn = Vue.extend({
    template:'#cpn2',
    props:['cmovies','cmessage']
})

在调用组件的标签内绑定父组件传过来的数据：

<div id="app">
    <cpn v-bind:cmovies="movies" :cmessage="message"></cpn>
</div>

这样子组件内就能接收到父组件的数据了：

<template id="cpn2">
    <div>
        <h2>我是子组件内容</h2>
        <p v-for="item in cmovies">得到父组件数据电影名称--->{{item}}</p>
        <p>得到父组件数据短信--->{{cmessage}}</p>
    </div>   
</template>
props用对象接收数据

用这种方法，还需要对传过来的数据进行一个验证，对传过来的数据有一个类型限制，还可以设置默认值default，还有如果required:true，调用的时候必须绑定这个属性

props:{
    cmovies:{
        type:Array,
    },
    cmessage:{
        type:String,
        default:'aaaaaa',
        required:true
    }
}
子传父：通过事件向父组件发送消息

举个例子，从子组件传递子组件的点击事件回到父组件，并且获得点击的id

首先在子组件定义一个点击事件：

@click="btnClick(item)"

<template id="cpn2">
    <div>
        <button v-for="item in categories" @click="btnClick(item)">{{item.name}}</button>
    </div>   
</template>
在子组件的方法里发送给父组件：

this.$emit('itemclick',item.id)

注意这里自定义的方法名不能用驼峰命名

methods:{
    btnClick(item){
        //把点击事件传给父组件
        this.$emit('itemclick',item.id)
    }
}
父组件的标签上接收这个事件

@itemclick="cpnClick"

<div id="app">
    <cpn @itemclick="cpnClick"></cpn>
</div>
在父组件的方法里就可以得到子组件传来的数据了
methods:{
    cpnClick(e){
        console.log(e);     //输出的是点击的子组件的id
    }
}
